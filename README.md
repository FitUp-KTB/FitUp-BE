# FitUp-BE

### 1. 커밋 컨벤션 (Commit Convention)
커밋 메시지는 다음 형식을 따른다

```
[타입]  커밋 내용
```

#### 1.1 커밋 타입
| 타입       | 설명 |
|------------|--------------------------------|
| `feat`     | 새로운 기능 추가 |
| `fix`      | 버그 수정 |

#### 1.2 커밋 메시지 예시
```
git commit -m "[feat]  API 기능 구현"
git commit -m "[fix] API 기능 수정"
```

### 2. 브랜치 전략
#### 2.1 브랜치 구조
- main : 운영 브랜치
- Feature 브랜치
- Fix 브랜치
```
feat/기능명/#이슈번호 : 기능 개발 브랜치
fix/버그명/#이슈번호 : 버그 수정 브랜치
```
